SEA Startup — Proyecto Web

Equipo: Alexandra · Sebastián · Elkin
Asignatura: Ingeniería Web
Entregable: Actividad práctica — HTML semántico y CSS

¿Qué construimos?

Desarrollamos SEA Startup, un sitio web que presenta una plataforma enfocada en ayudar a empresas y personas en Colombia a crear su presencia digital.

El proyecto fue construido utilizando HTML5 semántico y CSS, aplicando buenas prácticas de estructura web y diseño responsive. El sitio incluye información del proyecto, los perfiles del equipo, contenido académico sobre desarrollo web y diferentes componentes visuales que demuestran el uso de selectores y metodologías de diseño modernas.

Además, cada integrante del equipo desarrolló una página personal tipo blog de aprendizaje, donde se explican conceptos técnicos vistos durante el curso.

Explicación del código

El archivo index.html está organizado utilizando etiquetas semánticas de HTML5 para mantener una estructura clara y comprensible.

Estructura principal
<header>

Contiene el encabezado del sitio, incluyendo el logo del proyecto y el menú de navegación que permite acceder a las diferentes secciones.

<main>

Agrupa todo el contenido principal del sitio web.

Dentro de este elemento se encuentran varias secciones:

<section id="inicio">

Presenta la idea principal del proyecto y explica el objetivo de SEA Startup.

<section id="historia">

Describe el origen del equipo y cómo surgió la idea de crear la plataforma.

<section id="servicios">

Muestra los servicios o soluciones digitales que el proyecto busca ofrecer.

<section id="equipo">

Incluye los perfiles de los integrantes del equipo, cada uno con acceso a su página personal.

<section id="contenido-academico">

Contiene explicaciones relacionadas con ingeniería web, incluyendo temas como semántica HTML, selectores CSS y metodologías de diseño.

<footer>

Muestra información de derechos de autor, enlaces adicionales y créditos del equipo.

Páginas individuales del equipo

Cada integrante desarrolló una página personal donde se explican diferentes conceptos relacionados con el desarrollo web.

Estas páginas incluyen:

Una sección “Sobre mí” con información académica y áreas de interés.

Una explicación sobre la especificidad en CSS.

Una lista de selectores CSS que resultaron más interesantes o complejos durante el desarrollo.

Una galería de proyectos con imágenes que aplican efectos visuales usando la pseudo-clase :hover.

Una explicación de la metodología Mobile-First, utilizada para diseñar sitios adaptables a diferentes dispositivos.

Tecnologías utilizadas

En este proyecto utilizamos principalmente:

HTML5 semántico

CSS3

Selectores CSS

Pseudo-clases y pseudo-elementos

Media queries

Metodología Mobile-First

El objetivo fue aplicar estos conceptos para construir una estructura web clara, organizada y adaptable a diferentes tamaños de pantalla.

¿Qué aprendimos?

Este proyecto nos permitió comprender mejor cómo se construye la estructura de un sitio web utilizando HTML semántico y cómo los estilos en CSS influyen en la apariencia y el comportamiento visual de los elementos.

También aprendimos a trabajar con:

Especificidad en CSS, entendiendo cómo el navegador decide qué estilos aplicar.

Pseudo-clases como :hover, para crear interacciones visuales.

Pseudo-elementos como ::before y ::after, que permiten agregar contenido visual desde CSS.

Metodología Mobile-First, que prioriza el diseño para dispositivos móviles antes de adaptarlo a pantallas más grandes.

Dificultades que tuvimos

Una de las principales dificultades fue entender correctamente cómo funciona la especificidad en CSS, ya que en algunos casos los estilos no se aplicaban como esperábamos.

También tuvimos que ajustar varias veces la estructura del HTML para que fuera realmente semántica, utilizando correctamente etiquetas como section, article, header, main y footer.

Otra dificultad fue aplicar correctamente el enfoque Mobile-First, ya que requería pensar primero en pantallas pequeñas y luego adaptar el diseño para resoluciones mayores mediante media queries.

Conclusiones

Este proyecto nos permitió reforzar la importancia de escribir código limpio, estructurado y semántico. Aprendimos que la organización del HTML influye directamente en la claridad del proyecto y en la facilidad para mantener el código.

También comprendimos que CSS no solo se utiliza para decorar una página, sino que es una herramienta poderosa para controlar la interacción, la adaptabilidad y la experiencia del usuario.

Trabajar en equipo fue una parte importante del proceso, ya que nos permitió dividir responsabilidades y revisar el trabajo de los demás para mejorar el resultado final.
