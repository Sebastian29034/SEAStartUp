🎯 Especificidad en CSS

¿Qué es la especificidad?

La especificidad en CSS es el sistema que utiliza el navegador para decidir qué regla de estilo debe aplicarse cuando existen varias reglas que afectan al mismo elemento.

Cada selector tiene un nivel de prioridad. Por ejemplo, los selectores basados en ID tienen mayor peso que los selectores de clase, y estos a su vez tienen mayor prioridad que los selectores de etiquetas HTML como p, div o section.

Reflexión personal

Comprender este concepto fue importante durante el desarrollo del proyecto, ya que al principio no entendía por qué algunos estilos no se aplicaban correctamente.

Al aprender cómo funciona la especificidad, entendí que el navegador no solo sigue el orden del código, sino que también evalúa la prioridad de cada selector. Esto me ayudó a organizar mejor mi CSS y a evitar conflictos entre estilos.

🔍 Selectores que me parecieron atractivos

  • Selector de descendiente (div p): Permite aplicar estilos a elementos que se encuentran dentro de otros elementos. Es útil para trabajar con la jerarquía del HTML.
  • Pseudo-clase :hover: Permite cambiar el estilo de un elemento cuando el usuario pasa el cursor sobre él. Es muy utilizado para mejorar la interactividad de enlaces e imágenes.
  • Pseudo-elementos ::before y ::after: Permiten agregar contenido visual desde CSS sin modificar directamente el HTML.
  • Pseudo-clase :first-child: Selecciona el primer elemento dentro de un contenedor, lo cual es útil para aplicar estilos especiales.

📱 Metodología Mobile-First

¿Qué es Mobile-First?

Mobile-First es una metodología de diseño web que consiste en crear primero la versión del sitio para dispositivos móviles y posteriormente adaptar el diseño para pantallas más grandes como tablets y computadores.

Este enfoque permite priorizar el contenido más importante y garantizar que la página funcione correctamente en dispositivos con pantallas pequeñas.

Ventajas del enfoque

  • Mejor experiencia móvil: el diseño se optimiza primero para teléfonos.
  • Estructura progresiva: los estilos se agregan gradualmente según el tamaño de pantalla.
  • Mayor eficiencia: el sitio carga primero los estilos esenciales.

Aplicación en este proyecto

En este blog primero se diseñó la estructura para pantallas pequeñas organizando el contenido en una sola columna. Posteriormente se utilizó una media query con @media (min-width: 768px) para reorganizar algunos elementos en varias columnas en pantallas grandes.
