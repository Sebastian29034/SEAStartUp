🧠 Aprendizajes sobre jerarquía de estilos en CSS

¿Cómo decide el navegador qué estilo aplicar?

Cuando varias reglas de CSS afectan a un mismo elemento, el navegador necesita determinar cuál tiene prioridad. Para lograrlo utiliza un sistema de jerarquía conocido como especificidad, que permite calcular qué selector tiene mayor peso.

Este sistema funciona comparando diferentes tipos de selectores. Los selectores basados en ID tienen mayor prioridad, seguidos por clases, atributos y pseudo-clases, y finalmente los selectores de etiquetas HTML como div, p o section.

Reflexión personal

Comprender cómo funciona esta jerarquía me ayudó a organizar mejor mis estilos. Antes pensaba que el último estilo escrito siempre reemplazaba a los demás, pero en realidad el navegador evalúa la fuerza de cada selector.

Este conocimiento es importante porque evita conflictos en el diseño de una página. Cuando se usa CSS de forma estructurada y con selectores simples, el código se vuelve más fácil de mantener y también más claro para otros desarrolladores que trabajen en el mismo proyecto.

⚙️ Selectores CSS que me parecieron más interesantes

  • Pseudo-clase :first-child: Me llamó la atención porque permite seleccionar únicamente el primer elemento dentro de un contenedor. Es útil cuando se quiere resaltar el primer elemento de una lista o aplicar estilos especiales sin modificar el HTML.
  • Pseudo-clase :last-child: Funciona de manera similar al anterior, pero selecciona el último elemento dentro de un padre. Esto es muy útil para eliminar bordes o márgenes del último elemento de una lista o sección.
  • Pseudo-clase :hover: Permite modificar el estilo de un elemento cuando el usuario pasa el cursor sobre él. Se usa mucho para mejorar la interactividad en botones, enlaces y menús de navegación.
  • Selector de atributo ([target], [href]): Me pareció interesante porque permite seleccionar elementos según los atributos que tengan. Por ejemplo, se pueden estilizar enlaces que abren en otra pestaña usando a[target="_blank"].

📱 Diseño adaptable con enfoque Mobile-First

Concepto de Mobile-First

El enfoque Mobile-First consiste en desarrollar primero la interfaz pensando en dispositivos móviles, que tienen pantallas más pequeñas y recursos más limitados. A partir de esa base se agregan estilos adicionales mediante media queries para mejorar la distribución del contenido en pantallas más grandes.

Este enfoque permite construir sitios más eficientes, porque el diseño inicial se centra en lo esencial. Luego, a medida que aumenta el ancho de pantalla, se pueden incorporar elementos adicionales o reorganizar el contenido para aprovechar mejor el espacio disponible.

Ventajas principales

  • Mejor experiencia en móviles: Los teléfonos son actualmente el dispositivo más usado para navegar por internet, por lo que diseñar primero para ellos garantiza una interfaz clara y accesible.
  • Estructura más ordenada: El CSS se construye de forma progresiva, añadiendo estilos solo cuando son necesarios para resoluciones mayores.
  • Mayor adaptabilidad: El diseño se ajusta de forma gradual a tablets y computadoras, manteniendo la legibilidad y la organización del contenido.

Aplicación en esta página

En este proyecto se definieron primero los estilos base para pantallas pequeñas, organizando los elementos en una sola columna para facilitar la lectura en móviles. Posteriormente se utilizaron reglas con @media (min-width) para reorganizar algunos bloques de contenido en varias columnas cuando el sitio se visualiza en pantallas más amplias.
